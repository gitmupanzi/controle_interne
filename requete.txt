Améliorer les fichiers suivants :
- controle_interne\data\vision\requetes.sql

Je veux ajouter ou améliorer des requêtes de contrôle interne et d’aide à la décision à partir des tables et vues suivantes :

1. PRODUITS_EPG
Cette table contient les règles des produits d’épargne :
- dépôt autorisé ou non ;
- retrait autorisé ou non ;
- frais ;
- devise ;
- découvert ;
- intérêts ;

Cette vue doit être utilisée comme source principale des mouvements, car elle unifie :
- les mouvements back-office venant de HDPM ;
- les mouvements API/mobile venant de HDPM_API.

3. HDPM_API_VIEW
Cette vue peut être utilisée pour les analyses orientées API/mobile banking.

4. DEMANDES_CREDIT
Cette table contient les demandes de crédit :
- client demandeur ;
- agence ;
- produit crédit ;
- montant demandé ;
- état de la demande.

Attention très importante :
Dans DEMANDES_CREDIT, la colonne s’appelle exactement ID_POINT_SERIVCE.
Il y a probablement une faute dans le nom, mais il ne faut pas la corriger en ID_POINT_SERVICE si la colonne existe ainsi dans la base.
Utiliser exactement ID_POINT_SERIVCE dans les requêtes.

5. PRETS
Cette table contient les prêts accordés :
- montant ;
- dates ;
- devise ;
- comptes liés ;
- dossier de crédit ;
- source de financement.

6. CYCLES_PRET
Cette table contient les cycles ou échéances des prêts :
- date début ;
- date fin échéance ;
- date clôture ;
- montant ;
- prêt rattaché.

Intégrer les nouveaux contrôles dans la procédure stockée existante, sans casser les traitements déjà présents.

Contrôles à ajouter ou améliorer :

A. Produits d’épargne
- Produits inactifs encore utilisés par des comptes.
- Produits non valides utilisés par des comptes.
- Produits sans dépôt autorisé mais avec mouvements de dépôt.
- Produits sans retrait autorisé mais avec mouvements de retrait.
- Incohérences entre devise du produit, devise du compte et devise du mouvement.
- Comptes rattachés à un produit d’épargne inexistant ou invalide.

B. Mouvements HDPM_VIEW
- Mouvements sans compte.
- Mouvements avec compte inexistant.
- Mouvements sans opération rattachée.
- Mouvements avec montant nul, négatif ou anormal.
- Dépôts/retraits par client, compte, agence, devise et produit.
- Analyse des gros mouvements par période.
- Analyse des mouvements par point de service.

C. Crédit
- Demandes de crédit sans prêt accordé.
- Prêts sans dossier de crédit.
- Prêts sans compte de crédit.
- Prêts sans compte épargne lié.
- Prêts sans cycle dans CYCLES_PRET.
- Cycles de prêt avec échéance dépassée et non clôturée.
- Comparaison entre montant demandé dans DEMANDES_CREDIT et montant accordé dans PRETS.
- Analyse des crédits par agence, produit, devise et état.

D. Risques et décision
- Clients avec forte activité d’épargne et crédit actif.
- Clients avec plusieurs demandes de crédit sur une même période.
- Agences avec volume élevé de mouvements ou de crédits.
- Produits d’épargne les plus utilisés.
- Produits crédit les plus sollicités.
- Anomalies à prioriser pour audit.

Contraintes techniques :

- Garder une structure lisible.
- Ne pas supprimer les requêtes existantes sans justification.
- Ajouter des commentaires clairs avant chaque nouvelle requête.
- Si une jointure est incertaine, ajouter un commentaire au lieu d’inventer une relation.

CREATE VIEW [dbo].[HDPM_VIEW] 
AS 
SELECT ID as ID, DATE_OPERATION, DESCRIPTION, ID_OPERATION, LETTRE, MONTANT_OPERATION,
        NUM_TRANSACTION, NUMERO_RECU, RAPPROCHEMENT, SENS, VENTILATION, NUM_MOUVEMENT,

        DATE_VALEUR, NUM_LIGNE, MONTANT_REEL, REFERENCE_TIERS, ID_TIER
FROM hdpm

UNION

SELECT CODE as ID, DATE_OPERATION, DESCRIPTION, ID_OPERATION, LETTRE, MONTANT_OPERATION,
        NUM_TRANSACTION, NUMERO_RECU, RAPPROCHEMENT, SENS, VENTILATION, NUM_MOUVEMENT,
        ID_COMPTE, ID_DEVISE, ID_JOURNAL, ID_POINT_SERVICE, ID_TYPE_OPERATION, DATE_SAISIE,
        DATE_OPERATION as DATE_VALEUR, ID as NUM_LIGNE, MONTANT_OPERATION as MONTANT_REEL,

FROM hdpm_api
